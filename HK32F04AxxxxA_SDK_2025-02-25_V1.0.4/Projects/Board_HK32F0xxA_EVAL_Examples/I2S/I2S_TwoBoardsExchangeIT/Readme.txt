******************************例程详细描述*****************************
功能描述:
1. 此示例提供了一个小型应用程序，其中按钮用于使用中断并通过使用I2S固件库来触发I2S通信；
2. 此应用需要使用两块HK32应用板（A和B板），A板作为master，B板作为slave，连线方式如下：
*------------------------------------------------------------------------------*
|                BOARD A                                BOARD B                |
|         ____________________                   ____________________          |
|        |                    |                 |                    |         |
|        |                    |                 |                    |         |
|        |     __________     |                 |     __________     |         |
|        |    | SPI Master | ____ | _______SD________ | ____ | SPI Slave |    |         |
|        |    |  Device  | ____ | _______WS________ | ____ |  Device  |    |         |
|        |    | __________ | ____ | _______CK________ | ____ | __________ |    |         |
|        |                    |                 |                    |         |
|        |                    |                 |  O LD1             |         |
|        |            TAMPER  |                 |  O LD2             |         |
|        |               _    |                 |  O LD3             |         |
|        |              | _ |   |                 |  O LD4             |         |
|        |                    |                 |                    |         |
|        |             GND O-- | ---------------- - | --O GND             |         |
|        | ____________________ |                 | ____________________ |         |
|                                                                              |
|                                                                              |
*------------------------------------------------------------------------------ *
3. 这两个设备都是飞利浦标准配置，具有16位扩展到32位的数据包和48KHz的音频。
4. I2S发送中断和I2S接收中断都被启用。在这些中断子程序中，I2S_Buffer_Tx被传输，
并且接收到的值被加载到I2S_Buffer_Rx缓冲器中。只有有效的16个MSB被发送和接收，
而剩下的32个数据包的16个LSB被0值填充，并且不产生任何中断。
5. 当用户按下主板上的TAMPER按钮时，主发送器开始发送存储在I2S_Buffer_Tx缓冲区中的数据，
而从板接收数据I2S_Buffer_Rx缓冲区。
6. 传输完成后，会进行比较，TransferStatus会给出数据传输状态：
- 如果传输和接收的数据相同，则TransferStatus PASSED：绿色LED亮起。
- 如果传输和接收的数据不同，则传输状态失败：红色LED亮起。

时钟配置:
1. 本例程采用HSI56MHz作为系统时钟，用户如果需要更改系统时钟配置，
需要配置修改SYSCLK_SOURCE宏定义的。
2. 关于系统时钟如何配置，请参考对应用户手册或者RCC库函数及RCC相关例程。

适用芯片:
HK32F04AxxA

涉及外设:
I2S
GPIO

涉及管脚:
I2S1_WS - PA4
I2S1_CK - PA5
I2S1_SD - PA7
PC10 - LED1
PC12 - LED3
PC13 - BUTTON_TAMPER

注意事项:
1. 推荐使用MDK5.00及其以上版本编译本工程（MDK - ARM）
或者IAR8.22及其以上版本编译本工程(EWARM);
2. 使用之前检测电源VCC、GND等是否正常。

如何使用:
1. 直接编译程序下载到评估版，重启评估版;
