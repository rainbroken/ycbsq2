******************************例程详细描述*****************************
功能描述:
此示例显示如何定期更新WWDG计数器以及如何
模拟软件故障，在窗口看门狗到期时生成MCU WWDG重置

WWDG超时设置为43.7ms，刷新窗口设置为80。
WWDG计数器在主程序无限循环中每33毫秒刷新一次，以
防止WWDG重置。
LED2也每33毫秒切换一次，指示程序正在运行。
EXTI线连接到PC13引脚，并配置为生成中断在信号的上升沿上。

EXTI Line用于模拟软件故障：一旦发生EXTI Line事件
发生时，按下Tamper按钮，将提供相应的中断。
在中断中，写入无效地址会生成Hardfault异常，在该异常中断中无法喂狗（WWDG计数器为未刷新）。
结果，当WWDG计数器下降到63时，WWDG复位发生。
如果生成WWDG重置，则在系统从重置恢复后，LED1点亮。
并伴随串口1相应调试信息输出
如果EXTI Line事件没有发生，WWDG计数器将无限期刷新
在主程序无限循环中，并且没有WWDG重置。

时钟配置:
本例程采用HSE48MHz作为系统时钟，用户如果需要更改系统时钟配置，
需要配置修改SYSCLK_SOURCE宏定义的。
关于系统时钟如何配置，请参考对应用户手册或者RCC库函数及RCC相关例程。

适用芯片:
HK32F04AxxA

涉及外设:
WWDG

USART1
USART1配置如下:
―波特率 = 115200
―字长 = USART_WordLength_8b
―停止位 = USART_StopBits_1
―奇偶校验 = USART_Parity_No
―硬件流控制被禁用(RTS和CTS信号)
―使能接收和发送

涉及管脚:
USART1 TX - PA9
USART1 RX - PA10

BUTTON_TAMPER - PC13

LED1 - PC10
LED2 - PC11

注意事项:
1. 推荐使用MDK5.00及其以上版本编译本工程（MDK - ARM）
或者IAR8.22及其以上版本编译本工程(EWARM);
2. 使用之前检测电源VCC、GND等是否正常。
如何使用:
1. 直接编译程序下载到评估板，重启评估板;
