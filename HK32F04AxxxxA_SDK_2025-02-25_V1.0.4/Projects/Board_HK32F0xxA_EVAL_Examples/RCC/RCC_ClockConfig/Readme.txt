******************************例程详细描述*****************************
功能描述:
这个例子展示了如何:
―配置以HSE为时钟源的锁相环作为系统时钟源
―使用CSS (Clock Security System)特性检测HSE故障
―MCO引脚输出系统时钟

出于调试目的，RCC_GetClocksFreq()函数用于检索当前不同片上
时钟的状态和频率。
您可以看到RCC_ClockFreq结构内容，它保存不同的频率片上时钟，
使用您的工具链调试器。

本例还处理了高速外部时钟(HSE)故障检测(称为时钟安全系统，CSS):
当HSE时钟消失(损坏或损坏)时断开外部石英)，HSE和PLL被禁用(但PLL没有变化配置)，
HSI被选择作为系统时钟源和中断(NMI)。是生成的。

在NMI ISR中，HSE和HSE就绪中断是启用的。一旦HSE时钟在RCC ISR例程中，
生成HSERDY中断，并恢复系统时钟被重新配置到之前的状态(在HSE时钟失效之前)。
您可以监视MCO引脚(PA.8)上的系统时钟。两个led通过延时功能定义的定时进行切换。

时钟配置:
本例程采用HSE作为系统时钟，用户如果需要更改系统时钟配置，
需要配置修改SYSCLK_SOURCE宏定义的。
关于系统时钟如何配置，请参考对应用户手册或者RCC库函数及RCC相关例程。

适用芯片:
HK32F04AxxA

涉及外设:
RCC

涉及管脚:
PA8

注意事项:
1. 推荐使用MDK5.00及其以上版本编译本工程（MDK - ARM）
或者IAR8.22及其以上版本编译本工程(EWARM);
2. 使用之前检测电源VCC、GND等是否正常。
